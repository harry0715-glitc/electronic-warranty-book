## ⏳ 目前待處理 / 最高優先

1. **Query → Admin 修改模式：root cause 已定位並修正（2026-07-10 晚，待部署實測）**

   **Root cause**：進入編輯模式的每條路都依賴「頁面載入後的第二次請求或跨頁 sessionStorage」，而這在 Apps Script 沙箱環境不可靠——
   - sessionStorage staging：Query 與 Admin 的沙箱 iframe 來源（googleusercontent 網域）不同，跨頁讀不到
   - `getWarrantyById` JSONP：`<script src>` 是跨站子資源，手機 / LINE WebView 擋第三方 cookie 時帶不到 Google session → 驗證失敗
   - 失敗只寫 `console.warn`，於是「靜靜停在建立頁」

   **修正（已寫入程式碼）**：
   - `Code.gs`：`renderAdminHtml_` 渲染當下（已驗證請求、有完整 Sheet 權限）就把整筆案件與 LINE 綁定狀態注入頁面（`initialEditRecordJson` / `initialEditBindingJson`）——編輯模式不再需要任何第二次請求
   - `Code.gs`：新增 `adminRpc()`；AdminApp / QueryApp 的 API 傳輸改走 `google.script.run` 原生 RPC（身分隱含傳遞、不吃 cookie），JSONP 降為備援——連帶修正手機上綁定查詢與資訊卡發送的可靠性
   - `AdminApp`：編輯模式三層 fallback（注入資料 → staged → 即時 RPC），失敗改為 alert 可見，不再無聲

   **待做**：`clasp push` → 部署 B 更新新版本 → 手機 / LINE WebView 實測「查詢 → 修改」
